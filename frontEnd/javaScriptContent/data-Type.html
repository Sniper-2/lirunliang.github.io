    # 包装类型跟基本数据类型

    # 起因： 1 跟 new Number(1)的区别？

    • 两者的区别就是原始类型和包装对象的区别。

    # 基本数据类型跟包装对象的区分

    • 在JS中，数字、字符串、布尔、null、undefined以及ES6的Symbol被称之为基本数据类型的数据，其他数据类型都是对象，官方对于基本数据类型的定义是一种既非对象也无方法的数据。

    • 对象中有一类是Number, String, Boolean这三个对象，分别对应数字、字符串、布尔类型，我们称它们为包装对象或包装类型。

    # 基本数据类型的不可改变的特性

    • 基本数据类型一旦声明之后，那么它就将不可改变了，只能被替换。

    • 这个是因为JS对数据类型的存储方式的原因，在JS中，基本数据类型声明之后是存储在栈中，而不是堆中，所以当你对基本数据类型重新进行计算或者赋值的时候，实际上是重新声明了一个新的基本数据类型，再将这个基本数据类型赋值给了变量

    let test_a = 100;

test_a += 200;

console.log(test_a)   // 300
    • 上面的例子中 100 实际上还是存储在栈里面的，但是没有任何一个变量引用他，所以垃圾回收机制会在特定的时间对其进行回收，释放内存。

    # 基本类型的包装对象

    • 基本类型的包装对象是为了让开发者更好地操作基本数据类型而提供的内置对象；

    • 我们知道基本数据类型是没有属性跟方法的，然而我们声明一个字符串或者数字却能点出来它的某个方法，这个是因为我们在调用基本数据类型的方法的时候，js在后台或创建一个对应的数据类型的包装对象实例，再去调这个实例的方法，并返回结果，最后销毁实例；

    let test_a = 'lirunliang';        // 声明一个基础数据类型，再栈中保存该值；
let test_b = test_a.charAt(0);    // 调用了基础数据类型的某个方法，触发了js机制，所以实际上现在js后台执行的是一下代码：
{
  let _str = new String(test_a);  // 传入基础数据类型的值。实例化一个string对象
  test_b =  _str.chaAt(0);        // 调用该对象的方法，并返回值（返回值存放在栈里，赋值给test_b）
  _str = null;                    // 销毁实例
}

typeof test_a   // string,数据类型没有发生改变
typeof test_b   // string,数据类型没有发生改变
