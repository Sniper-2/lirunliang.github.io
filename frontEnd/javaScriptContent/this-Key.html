    # this关键字的指向问题

    跟其他语言相比，this关键字在JS中的表现有所不同，是否是严格模式也会影响this的绑定，在绝大多数情况下，函数的调用方式决定了this的值。this不能在执行期间被赋值，并且在每次函数被调用时this的值也可能会不同。

    # 1.JS的引用类型

    为什么有this这个关键字呢，首先要知道JS的引用数据类型；

    JS的引用类型有Object、Array、Function等Object type的数据；

    引用类型声明之后，是保存在堆内存之中的，由于JS不可以直接访问堆内存空间中的位置和操作堆内存空间，所以每当我们声明一个引用类型的数据的时候，实际上是将该数据在堆中的地址(所谓的指针)保存到栈中，再将栈中的地址指向引用数据类型的变量名，所以我们每次调用一个对象、数组、函数等引用类型的数据时，实际上是先查找该数据在栈中的位置，再去堆中拿到对应的数据结构内容，再执行；

    let dataObj = {
  name: '李润亮',
  testFun: function () {
    console.log(this)
  }
}

    # 2.this的指向

    知道引用类型的存储方式之后，就能够更加容易理解this的指向的问题了；

    # 第一个例子

    var test1 = {
  name: '李润亮',
  testFun: function () {
    return '姓名：'+ this.name;
  }
};

var test2 = {
  name: '李润亮人造人二号'
};

test2.testFun = test1.testFun;
test2.testFun()

// 姓名：李润亮人造人二号

    结果显而易见，我们学了JS之后这种简单的引用关系基本上一眼就能看出来，但是最主要的还是要知道内存里面是如何对上面的对象结构进行存储跟引用的
    上面的例子实际上就是声明了一个函数之后，会在栈生成一个该函数存放在堆里面的引用地址，然后把引用地址赋值给testFun这个变量
    最后自动绑定testFun指向的this 重构一下:

    var test1 = {
  name: '李润亮',
  testFun: f      // 函数在栈内存中的引用地址
};


// f 为引用地址 ，函数存在堆内存中
function f () {
    return '姓名：'+ this.name;
}

var test2 = {
  name: '李润亮人造人二号'
};


/**
 * 因为在 test1 里面的 testFun 属性实际上是存储了 f 函数的引用地址；
 * 现在把引用地址告诉 test2.testFun ，
 * 然后 test2.testFun 跟 test1.testFun 都指向了存放在栈中的 f 函数引用地址
 */
test2.testFun = test1.testFun;

    # 3.this的使用场合

    this有三种使用场景

    • 全局环境
      全局环境使用this，它指的就是顶层对象window.

    • 构造函数
      构造函数中的this，指的是实例对象

      我们知道构造函数最后都是返回一个实例对象，所以在构造函数内部的this实际是指的是这个返回回去的实例对象；

      function Obj = function (test) {
  this.a = test
}

let myObj = new Obj('李润亮');
myObj.a // -> 李润亮

      new的时候实际上跟this的指向也是息息相关；
      我们在执行new关键字的时候，内部是需要经过5个步骤，最后才返回实例对象的:

      • 1.创建一个空对象，作为将要返回的对象实例
      • 2.将这个空对象的原型，指向构造函数的prototype属性，形成一条完整的原型链
      • 3.将函数内部的this指向这个空对象
      • 4.开始执行构造函数内部的代码
      • 5.返回这个对象或者构造函数返回的对象

      上面的代码我们看用重构为:

      // 
/**
 * new关键字代码逻辑
 * constructor 构造函数
 * params      实例化构造函数传的参数
 */
function new (constructor, params) {
  // 拿到构造函数跟实例化的参数
  let args = [].slice.call(arguments);

  // 取出构造函数
  let constructor = args.shift();

  // 创建一个空对象，继承构造函数的 prototype 属性，形成完整的原型链
  let context = Object.create(constructor.prototype);

  // 执行构造函数，构造函数内部的this指向我们创建出来的空对象
  let result = constructor.apply(context, args);

  // 返回，如果我们本身的构造函数已经返回了一个对象，那么就执行构造函数的返回，否则返回我创建的对象
  return (typeof result === 'object' && result != null) ? result : context;
}


// 进行实例化
function Obj = function (test) {
  this.a = test
}

let myObj = new Obj('李润亮');
myObj.a // -> 李润亮

// ----- ↓↓↓↓↓ new的时候执行new方法，变成了 ↓↓↓↓ -----
new(Obj, '李润亮') {
  // args = [Obj, '李润亮']
  let args = [].slice.call(arguments);

  // constructor == Obj
  let constructor = args.shift();   
        
  // context = {}   context.prototype === Obj.prototype
  let context = Object.create(constructor.prototype);

  /**
   * 执行构造函数
   * 等于 Obj(args).bind(context)
   * Obj内部的this == context
   * 所以Obj代码执行 this.a = test 等于 context.a = test
   * 而test等于我们传进去的"李润亮", 所以 context.a = "李润亮"
   */
  let result = constructor.apply(context, args);

  // 因为我们的构造函数Obj没有返回一个对象，所以我们把context返回回去
  return (typeof result === 'object' && result != null) ? result : context;
}

      实际上这也涉及到引用数据类型在内存中存储的知识
      我们在上面有一段代码，绑定构造函数的this指向

      let result = constructor.apply(context, args);

      实际上我们在创建context的时候已经在栈中生成了一个指向该对象在堆中地址的引用地址
      构造函数内部也的this也绑定了这个引用地址，所以我们在构造函数内部修改该引用地址的对象的属性后 在new函数里面的context也会一起更改,因为他们指向了同一个引用地址

    • 对象的方法 对象里面的方法的this的指向就是方法运行时所在的对象。该方法赋值给另一个对象，就会改变this的指向。 举个例子

      • 第一个
      let obj ={
  foo: function () {
    console.log(this);
  }
};
(obj.foo = obj.foo)() // -> window

      一步一步解析 重构上面的例子

      function foo () {
  console.log(this);
}
let obj ={
  foo: foo
};

// 输出obj.foo，会得到一个函数(而不是foo函数的执行结果)
console.log(obj.foo) // function () { console.log(this); }

// 那么 (obj.foo = obj.foo)() 就等于
(obj.foo = function () { 
  console.log(this)
)()

// 因为()()是一个自执行的函数，会回去执行第一个括号里面的代码并得出返回值，那么
// (obj.foo = function () {  console.log(this) )() 继续执行第二🙅步，等于
(obj.foo)()

// 然后我们在刚开始已经输出过obj.foo的值，这个值等于 function () { console.log(this); }
// 对象把方法点出来而不执行，上面的代码等于
(function () { 
  console.log(this); 
})()

// 最后生成了一个自执行函数
// 并且我们知道 对象里面的方法的this的指向就是方法运行时所在的对象
// 所以上面的代码的this是指向顶层对象，就是window
(function () { 
  console.log(this); 
})()

// 所以
(obj.foo = obj.foo)() // -> window

      # 4.this在对象方法中使用的作用域

      当this所在的方法不在对象的第一层，这时this只是指向当前一层的对象，而不会继承更上面的层。

      # 例子
      let test = {
  name: '李润亮',
  action: {
    actionA: function() {
      console.log(this.name);
    }
  }
};
test.action.actionA()  // undefined

      这个也是涉及到引用引用类型的数据再内存储的问题；
      上面的代码实际上是这样的：

        let test = {
    name: '李润亮',
    action: {
      actionA: function() {
        console.log(this.name);
      }
    }
  };

  let action = {
    actionA: actionA
  }

  function actionA () {
    console.log(this.name);
  }

      在action对象声明的时候会把actionA的引用地址赋值给action.actionA
      之后在action对象调用action.actionA的时候会自动把action当成this绑定给actionA函数
      而action对象里面并没有name属性，所以是undefined
