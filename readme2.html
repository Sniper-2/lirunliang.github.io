# 关于本站

本站是基于VuePress搭建而成，部署在git pages上，从阿里云解析至自己的域名，下面将简单扼要地介绍一下VuePress的基础搭建与部署还有域名的解析。

# 开始

首先安装VuePress

  # 全局安装
  npm install -g vuepress

之后创建一个文件目录结构如下的文件夹, vuepress官方没看到初始化文件夹的指令，只能自己手动创建了。 或者先npm init y进行文件夹初始化也行的

|-- fileName
    |-- package-lock.json
    |-- package.json
    |-- README.md
    |-- docs
        |-- README.md
        |-- .vuepress
        |   |-- config.js

docs

docs 是文件根目录，docs下的README.md将在VuePress打包后作为整个网站的首页进行展现，例如你现在看到的这些内容，都是在这个MD文件内部中书写的。如果书写过程中有vue组件，也是从该根目录一层一层往内部进行查找的。

.vuepress

.vuepress 是必须的，这是放置所有 VuePress 特有(VuePress-specific) 文件的地方，VuePress将会从该文件夹内查找对应配置文件或Vue组件

.vuepress/config.js

.vuepress/config.js配置 VuePress 站点的基本文件, 其中导出一个JS对象，网站的导航以及路由匹配到的文件等等的配置都是需要从该文件中进行配置。

## 首页测试

fileName/docs/.vuepress/config.js 关于config.js的基本配置，你可以参考config基本配置

module.exports = {
  title: '为了达拉然!',              // 页面title
  description: 'this is my file',   //描述
  base: '',
  themeConfig: {
    serviceWorker: {
      updatePopup: true 
    },
    displayAllHeaders: false,
    lastUpdated: '更新时间',
    nav:[ // 导航栏配置
      {text: '主页', link: '/' },    
    ],
    sidebar: {},
    sidebarDepth: 2, // 侧边栏显示2级
  }
};

到了这一步，执行npm run dev项目就能正常运行起来了，其他基础而配置建议手动修改看结果。

# 打包部署到GitHub上面

  • 在github上面建立一个名为你的用户名.github.io的git仓库
  • 在本地执行npm run build进行打包
  • 进入docs/.vuepress/dist,该文件夹为打包后的项目，将该文件夹推到刚刚建立的git仓库中
  • 进入git仓库 -> 点击settings -> 滚动至GitHub Pages
  • GitHub Pages会展现你当前仓库项目的访问地址Your site is ready to be published at https://你的用户名.github.io/你的用户名.github.io/.
  • 点击该地址你就能正常看到
